# API Document Retrieval 技能

## 📋 技能概述

**API Document Retrieval** 是一个智能匹配技能，能够根据需求清单中的关键字，自动从 **T100与OA类系统集成标准API对照清单** 中匹配对应的API接口信息和字段映射。

### 数据源

本技能支持两种数据源（优先级从高到低）：

1. **本地文件**（优先）
   - 路径: `需求对照工具表/【2026】T100&OA类系统集成清单字段对照表.xlsx`
   - 优势: 离线可用、读取速度快、支持完整字段匹配
   
2. **在线文档**（备选）
   - 地址: https://docs.qq.com/sheet/DSmdhek1TT3BPenRa?tab=BB08J2
   - 优势: 实时更新、多人协作、最新版本
   - 注意: 需要网络连接和腾讯文档访问权限

---

## ✨ 核心能力

### 1. 智能关键字提取
- 自动识别业务对象（采购订单、销售订单、审批流程等）
- 提取字段名称（单号、状态、日期、金额等）
- 识别系统标识（T100、OA、ERP等）
- 识别操作类型（创建、更新、审批、同步等）

### 2. 多维度API匹配
- **高匹配度**：业务对象精确匹配 + 字段匹配
- **中匹配度**：场景匹配 + 类型匹配
- **低匹配度**：关键词匹配 + 系统匹配

### 3. 生成详细报告
- 匹配度评分和排序
- API接口详细信息
- 字段映射对照表
- 转换规则说明
- 源文件位置引用（工作表、行号）

## 🚀 快速开始

### 前置要求

```bash
# 安装依赖
pip install openpyxl python-docx
```

### 使用步骤

#### 1. 准备需求清单
支持以下格式：
- `.docx` - Word 文档
- `.xlsx` / `.xls` - Excel 文件
- `.txt` - 文本文件
- `.md` - Markdown 文件

#### 2. 运行匹配脚本

```bash
python .qoder/skills/api-document-retrieval/scripts/match_api.py <需求清单文件路径>
```

**示例**：
```bash
python .qoder/skills/api-document-retrieval/scripts/match_api.py "各地区集成需求/上海中柔_OA抛转调拨单接口需求确认书.docx"
```

#### 3. 查看匹配报告

脚本会生成 `API匹配报告_YYYYMMDD.md` 文件，包含：
- 提取的关键字列表
- 匹配度统计
- 高/中/低匹配度的API接口
- 字段映射详情
- 参考位置指引

## 📊 匹配示例

### 输入需求
```
客户需求：OA审批流程对接T100创建采购订单

需要字段：
- 采购申请单号
- 申请人
- 申请日期
- 物料编码
- 采购数量
- 总金额
- 审批状态
```

### 提取关键字
- **业务对象**: 采购订单、审批流程
- **关键字段**: 单号、申请人、日期、金额、状态
- **涉及系统**: OA、T100
- **操作类型**: 创建、审批

### 匹配结果
```
🔴 高匹配度: 2 个
🟡 中匹配度: 3 个
🟢 低匹配度: 5 个
```

### 输出报告节选
```markdown
## 🔴 高匹配度 API

### 1. API 接口 (匹配度: 95%)

**参考位置**: 工作表 `采购接口` | 行号 `45-67`
**匹配依据**: 业务对象: 采购订单, 字段: 单号, 系统: OA, 动作: 创建

#### 字段映射
| 源字段 | 目标字段 | 类型 | 转换规则 |
|--------|---------|------|---------|
| docNo | docno | String→varchar(30) | 直接映射 |
| status | status | String→char(1) | Approved→Y |
| amount | total_amt | Double→decimal(15,2) | 保留2位小数 |
```

## 💡 提高匹配准确度

### ✅ 好的做法
1. **详细描述业务场景**
   ```
   ✅ "在OA中发起采购申请，审批通过后自动在T100创建采购订单"
   ```

2. **列出关键字段**
   ```
   ✅ "字段：单号、申请人、日期、物料、数量、金额、状态"
   ```

3. **明确接口方向**
   ```
   ✅ "OA → T100" 或 "T100同步到OA"
   ```

4. **使用标准术语**
   ```
   ✅ 采购订单、审批流程、抛转、同步
   ```

### ❌ 避免的做法
1. **描述过于简单**
   ```
   ❌ "采购接口"
   ```

2. **缺少字段信息**
   ```
   ❌ "需要字段映射"（未列出具体字段）
   ```

3. **方向不明确**
   ```
   ❌ "T100和OA对接"（未说明方向）
   ```

## 🔧 故障排除

### 问题 1: 脚本执行失败
**错误**: `ModuleNotFoundError: No module named 'openpyxl'`

**解决**:
```bash
pip install openpyxl python-docx
```

### 问题 2: 找不到匹配结果
**原因**: 
- 需求描述不够详细
- 关键字提取失败
- 对照清单文件不存在

**解决**:
- 补充业务场景和字段描述
- 检查文件路径是否正确
- 确认 `需求对照工具表/【2026】T100&OA类系统集成清单字段对照表.xlsx` 存在
- 如果本地文件不存在，访问在线文档: https://docs.qq.com/sheet/DSmdhek1TT3BPenRa?tab=BB08J2

### 问题 3: 匹配度都很低
**原因**:
- 需求描述与标准API术语不一致
- 缺少关键字段信息

**解决**:
- 使用标准业务术语
- 添加更多字段信息
- 参考 examples.md 中的示例

### 问题 4: 本地文件不存在
**解决**:
1. 访问在线文档: https://docs.qq.com/sheet/DSmdhek1TT3BPenRa?tab=BB08J2
2. 点击右上角「下载」按钮
3. 选择 Excel 格式
4. 保存到: `需求对照工具表/`
5. 确保文件名: `【2026】T100&OA类系统集成清单字段对照表.xlsx`

### 问题 5: 在线文档无法访问
**解决**:
- 检查网络连接
- 确认腾讯文档账号登录状态
- 验证文档访问权限（可能需要申请权限）
- 联系文档管理员获取访问权限

## AI 辅助匹配（新增 v2.0）

### 功能概述

当本地规则匹配无法找到合适的接口时，可以使用 **AI 辅助匹配** 功能作为兜底方案。

AI 辅助匹配使用 **Kimi API**（moonshot-v1-32k 模型）进行语义检索匹配：
- 自动提取需求中的业务实体和动作
- 在接口清单中进行语义相似度分析
- 返回最匹配的接口推荐及理由

### 前置条件

1. **安装依赖**：
   ```bash
   npm install xlsx
   ```

2. **配置 Kimi API Key**：
   ```bash
   # 方式 1：环境变量（推荐）
   set KIMI_API_KEY=sk-your-api-key

   # 方式 2：命令行参数
   node match_ai_assist.js xxx.xlsx --api-key sk-your-api-key
   ```

   获取 API Key：访问 https://platform.moonshot.cn/ 注册并创建

### 使用方式

#### 失败兜底模式（默认）
只分析本地规则匹配失败的行（匹配度 < 40%）：
```bash
node .qoder/skills/api-document-retrieval/scripts/match_ai_assist.js 需求评估整理/AI输入摸板.xlsx
```

#### 全量分析模式
对所有需求行都进行 AI 分析：
```bash
node .qoder/skills/api-document-retrieval/scripts/match_ai_assist.js 需求评估整理/AI输入摸板.xlsx --all
```

#### 高级选项
```bash
# 调整匹配阈值
node match_ai_assist.js xxx.xlsx --threshold 50

# 指定输出目录
node match_ai_assist.js xxx.xlsx --output 自定义输出目录/

# 指定模型
node match_ai_assist.js xxx.xlsx --model moonshot-v1-8k

# 查看所有选项
node match_ai_assist.js --help
```

### 输出文件

运行后会生成两个文件：

1. **AI辅助匹配报告** (`xxx_AI辅助匹配报告.md`)
   - 匹配统计（本地成功 / AI成功 / 仍未匹配）
   - AI 推荐接口详情（流程名称、匹配确信度、匹配理由）
   - 仍未匹配的需求列表

2. **JSON 数据文件** (`xxx_ai_matches.json`)
   - 每条需求的本地匹配结果和 AI 匹配结果
   - 便于后续程序化处理

### 与 V6 配合使用

推荐的工作流程：
```bash
# 1. 先运行 V6 本地规则匹配
node .qoder/skills/api-document-retrieval/scripts/match_nenglv_v6.js 需求评估整理/AI输入摸板.xlsx

# 2. 再运行 AI 辅助匹配，对失败行进行兜底分析
node .qoder/skills/api-document-retrieval/scripts/match_ai_assist.js 需求评估整理/AI输入摸板.xlsx

# 3. 查看两份报告，人工复核 AI 推荐结果
```

### 注意事项

1. **API 调用成本**：每次分析会消耗 Kimi API 的 Token，建议只对匹配失败的行使用
2. **网络要求**：需要稳定的网络连接访问 Kimi API
3. **结果复核**：AI 推荐结果仅供参考，建议人工确认后再采用
4. **Rate Limit**：脚本已实现自动重试和请求间隔，避免触发 API 限制
