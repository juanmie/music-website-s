# V2 智能匹配增强版使用说明

## 📋 功能概述

**API Document Retrieval V2** 是专门为 Excel 格式需求清单设计的智能匹配工具。

### 核心能力

1. ✅ **智能识别开发分类**
   - 识别 `710:客户端_推送接口`
   - 自动确定服务方为 T100
   - 自动设置流向为 `T100→XX系统`

2. ✅ **智能识别客制类型**
   - 识别 `新增` 类型
   - 自动匹配复用接口

3. ✅ **高度匹配算法**
   - 程序代号 + 流向 匹配
   - 对照接口清单的"流程名称"
   - 匹配度 ≥70% 认为成功

4. ✅ **自动追加结果**
   - 在原需求清单中追加匹配列
   - 自动备份原文件
   - 包含完整参考链接

## 🚀 快速开始

### 1. 准备需求清单

需求清单 Excel 文件应包含以下列（列名可略有差异）：

| 必需列 | 示例值 | 说明 |
|--------|--------|------|
| 开发分类 | `710:客户端_推送接口` | 触发匹配的关键条件 |
| 客制类型 | `新增` | 表示需要匹配复用接口 |
| 程序代号 | `AXCP001` | 用于匹配流程名称 |
| 流向 | `T100→OA` | 接口方向（可选，自动推断） |

**示例需求清单结构**：

```
| 需求编号 | 需求名称 | 开发分类 | 客制类型 | 程序代号 | 流向 |
|---------|---------|---------|---------|---------|------|
| REQ001 | OA审批推送 | 710:客户端_推送接口 | 新增 | AXCP001 | |
| REQ002 | 物料同步 | 710:客户端_推送接口 | 新增 | IXM001 | |
```

### 2. 运行匹配脚本

```bash
python .qoder/skills/api-document-retrieval/scripts/match_api_v2.py <需求清单Excel文件>
```

**示例**：

```bash
python .qoder/skills/api-document-retrieval/scripts/match_api_v2.py "集成需求清单产出/上海中柔_OA抛转调拨单接口需求确认书_标准化.xlsx"
```

### 3. 查看结果

脚本会生成两个文件：

1. **备份文件**: `xxx_备份_YYYYMMDD_HHMMSS.xlsx`
   - 原始需求清单的完整备份

2. **匹配结果**: `xxx_已匹配.xlsx`
   - 在原需求清单基础上追加匹配列

## 📊 匹配规则详解

### 规则 1: 开发分类识别

**触发条件**：
```
开发分类 包含 "710" 且 包含 "推送接口"
```

**自动操作**：
- 设置服务方 = T100
- 设置流向 = `T100→外部系统`（如果流向为空）

**示例**：
```
开发分类: "710:客户端_推送接口"
→ 服务方: T100
→ 流向: T100→OA
```

---

### 规则 2: 客制类型识别

**触发条件**：
```
客制类型 包含 "新增"
```

**自动操作**：
- 启动复用接口匹配流程
- 在接口清单中查找可复用接口

**示例**：
```
客制类型: "新增"
→ 需要匹配现有接口
→ 避免重复开发
```

---

### 规则 3: 高度匹配逻辑

**匹配字段**：
```
需求清单: 程序代号 + 流向
对照清单: 接口清单页签 → 流程名称
```

**匹配度计算公式**：

```
总匹配度 = 程序代号匹配(50%) + 流向匹配(30%) + 关键词匹配(20%)
```

#### 程序代号匹配（50分）

```python
if 程序代号 in 流程名称:
    score += 50
```

**示例**：
```
程序代号: "AXCP001"
流程名称: "AXCP001-OA审批推送接口"
→ 匹配得分: 50分
```

#### 流向匹配（30分）

```python
if 流向 in 流程名称:
    score += 30
elif "T100" in 流程名称 and "T100→" in 流向:
    score += 20
```

**示例**：
```
流向: "T100→OA"
流程名称: "T100→OA审批接口"
→ 匹配得分: 30分
```

#### 关键词匹配（20分）

```python
关键词 = ["推送", "接口", "同步", "创建", "查询"]
for 关键词 in 关键词列表:
    if 关键词 in 程序代号 and 关键词 in 流程名称:
        score += 10
        break
```

**示例**：
```
程序代号: "推送接口001"
流程名称: "OA推送接口"
→ 匹配得分: 10分（"推送"匹配）
```

---

### 规则 4: 结果追加

**匹配成功后**，自动在需求清单中追加以下列：

| 列名 | 说明 | 示例值 |
|------|------|--------|
| 匹配状态 | ✅ 已匹配 / ❌ 未匹配 | `✅ 已匹配` |
| 匹配接口 | 接口名称 | `CreatePurchaseOrder` |
| 流程名称 | 对照清单中的流程名称 | `AXCP001-OA审批推送接口` |
| 接口方向 | 接口流向 | `T100→OA` |
| 匹配度 | 百分比 | `85%` |
| 匹配依据 | 匹配原因说明 | `程序代号:AXCP001, 流向:T100→OA → 流程名称:AXCP001-OA审批推送接口` |
| 参考位置 | 工作表和行号 | `接口清单 行45` |
| 在线文档 | 在线文档链接 | `https://docs.qq.com/sheet/...` |

## 🔧 高级配置

### 调整匹配度阈值

编辑 `match_api_v2.py` 第 169 行：

```python
if match_score >= 70:  # 修改此值调整阈值
```

- **降低阈值**（如 60）：更多匹配，但可能不准确
- **提高阈值**（如 80）：更准确，但可能漏掉一些

### 调整匹配权重

编辑 `calculate_match_score` 方法（第 179-203 行）：

```python
# 程序代号匹配权重（默认50）
if prog_code and prog_code in flow_name:
    score += 50  # 调整此值

# 流向匹配权重（默认30）
if flow_dir and flow_dir in flow_name:
    score += 30  # 调整此值
```

### 添加新的关键词

编辑第 197 行：

```python
keywords = ['推送', '接口', '同步', '创建', '查询', '你的关键词']
```

## 🎯 最佳实践

### ✅ 推荐做法

1. **规范需求清单格式**
   - 使用标准列名
   - 填写完整的开发分类和客制类型
   - 提供准确的程序代号

2. **定期更新接口清单**
   - 保持本地文件最新
   - 确保接口清单完整准确

3. **检查匹配结果**
   - 人工复核高匹配度结果
   - 处理未匹配的需求

4. **保留备份文件**
   - 自动生成的备份不要删除
   - 便于回溯和对比

### ❌ 避免做法

1. **不规范的列名**
   - ❌ "分类"、"类型"（太模糊）
   - ✅ "开发分类"、"客制类型"（明确）

2. **缺失关键信息**
   - ❌ 不填写程序代号
   - ✅ 填写准确的程序代号

3. **忽略匹配结果**
   - ❌ 不查看匹配报告
   - ✅ 人工复核匹配结果

## 🔍 故障排除

### 问题 1: 找不到匹配结果

**原因**：
- 程序代号不准确
- 接口清单中不存在对应接口
- 匹配度过低

**解决**：
- 检查程序代号是否正确
- 查看接口清单是否有对应接口
- 降低匹配度阈值（临时）

### 问题 2: 匹配度都很低

**原因**：
- 程序代号命名不一致
- 流程名称格式不统一

**解决**：
- 统一命名规范
- 调整匹配权重
- 添加更多关键词

### 问题 3: 脚本执行失败

**错误**: `ModuleNotFoundError: No module named 'openpyxl'`

**解决**:
```bash
pip install openpyxl
```

**错误**: `找不到接口清单页签`

**解决**:
- 检查 Excel 文件是否有"接口清单"页签
- 或使用其他包含接口数据的页签
