# API Document Retrieval

根据需求清单关键字，自动从 T100与OA类系统集成标准API对照清单 中匹配对应的API接口和字段映射信息。

## 数据源

本技能基于以下两份数据源进行检索（优先级从高到低）：

### 1. 本地文件（优先）
- **字段对照清单**: `需求对照工具表/【2026】字段对照---最小颗粒度.xlsx`
  - 内容：每个工作表对应一个流程的详细字段对照（接收方/发起方字段、参数类型、赋值逻辑等）
  - 结构：工作表名包含程序代号（如 axmt540、aimm200）
- **接口清单汇总**: `需求对照工具表/【2026】接口清单---汇总.xlsx`
  - 内容：所有接口的汇总列表，包含流程名称、接口方向、功能描述、适配类型等
  - 结构：按系统分类（OA接口清单、WMS接口清单、MES接口清单）
- **优势**: 离线可用、读取速度快、支持完整字段匹配
- **使用**: 自动读取本地 Excel 文件

### 2. 在线文档（备选）
- **接口清单汇总在线文档**: https://docs.qq.com/sheet/DWmF4RXZmdW5RWnVk?tab=nxp5uc
- **字段对照清单在线文档**: https://docs.qq.com/sheet/DWm9PT0xtYWdYRW5R?scene=a815f6149367bdcef5a2dc6d3cXfw1&tab=mcy037
- **优势**: 实时更新、多人协作、最新版本
- **使用**: 当本地文件不存在或需要最新数据时访问
- **注意**: 需要网络连接和腾讯文档访问权限
- **下载方式**:
  - 手动下载（推荐）：浏览器打开 → 点击下载 → 保存
  - Cookie 下载：`python scripts/download_online_doc.py --cookie '<cookie>'`
  - 浏览器自动化：需要安装 Playwright（不推荐，维护成本高）

**数据源选择策略**：
1. 优先使用本地文件（速度快、离线可用）
2. 本地文件不存在时，提示用户访问在线文档
3. 需要最新数据时，可手动从在线文档下载更新本地文件

## 工作流程

### 版本说明

本技能提供两个版本的匹配脚本：

**V1 - 关键字匹配版** (`match_api.py`)
- 适用场景：从文本/docx/txt中提取关键字匹配
- 匹配方式：基于业务对象、字段、系统关键字
- 输出：生成独立的 Markdown 报告

**V2 - 智能匹配增强版** (`match_api_v2.py`)
- 适用场景：处理 Excel 格式的需求清单
- 匹配方式：基于开发分类、客制类型、程序代号、流向
- 输出：直接在原需求清单中追加匹配结果

**V5 - 能率需求清单智能匹配（格式完整保留版）** (`match_nenglv_v5.js`) ✨ **推荐**
- 适用场景：处理能率/国家平台集成需求清单（.xlsx格式）
- 匹配方式：基于开发分类自动推断流向 + 程序代号+流向匹配流程名称
- 数据源：
  - 匹配数据源：`【2026】接口清单---汇总.xlsx`（OA/WMS/MES接口清单汇总）
  - 字段对照源：`【2026】字段对照---最小颗粒度.xlsx`（每个工作表对应一个流程的字段对照）
- 技术栈：Node.js + ExcelJS 库（完整保留边框、颜色、字体、合并单元格、列宽、行高）
- 输出：生成独立的 `_API匹配结果.xlsx` 文件
- 特点：
  - 自动推断流向：702/704→XX→T100, 710→T100→XX, 202→T100→XX
  - 智能匹配：程序代号精确匹配(60%) + 流向匹配(30%) + 关键词匹配(10%)
  - 支持 .xlsx 格式（推荐使用）
  - **完整保留格式**：边框、填充颜色、字体、合并单元格、列宽、行高、对齐方式
  - 匹配到的流程详情页签完整复制到输出文件，格式与源文件一致

**V4 - 能率需求清单智能匹配** (`match_nenglv_v4.js`)
- 旧版本，使用 xlsx 库，格式保留不完整（不推荐使用）

### V4 新增匹配规则（能率需求清单专用）

#### 规则 1: 开发分类自动推断流向
- **702/703/704/705**: 服务端接口 → 流向 = `XX→T100` (BPM/OA推送数据到T100)
- **710/711**: 客户端接口 → 流向 = `T100→XX` (T100推送数据到BPM/OA)
- **202**: 基本资料作业 → 流向 = `T100→XX` (T100提供数据查询接口)

#### 规则 2: 程序代号+流向匹配流程名称
- **匹配字段**: 程序代号 + 推断流向
- **对照字段**: 需求对照工具表 → 【2026】接口清单---汇总.xlsx → 各系统接口清单页签 → 流程名称
- **匹配度**: ≥40% 认为匹配成功
- **计算公式**:
  - 程序代号精确匹配: 60%
  - 程序代号包含匹配: 40%
  - 流向匹配: +30%
  - 流向相反: -20%
  - 业务关键词匹配: +10%
  - 规格说明提及: +5%

#### 规则 3: 匹配结果输出
匹配成功后，生成新的 Excel 文件，包含：

**工作表 1: 需求匹配结果**
- ✅ 匹配状态（已匹配/未匹配）
- 匹配流程名称
- 接口方向
- 推断流向
- 匹配度
- 匹配依据
- 参考位置（工作表+行号）
- 在线文档链接

**工作表 2~N: 匹配到的流程详情页签**
- 每个匹配成功的流程，都会从 `【2026】字段对照---最小颗粒度.xlsx` 中复制对应的字段对照工作表到输出文件中
- 包含：服务发起方/接收方、接口字段对照（入参/出参）、报文示例等详细信息
- 格式与源文件完全一致（边框、颜色、字体、合并单元格等）

## 匹配规则

### 高优先级匹配
1. **精确匹配**：接口名称完全一致
2. **业务对象匹配**：采购订单→PurchaseOrder API
3. **字段映射匹配**：关键字段名称匹配

### 中优先级匹配
1. **模糊匹配**：接口名称相似度 > 70%
2. **场景匹配**：相同业务场景（审批、同步等）
3. **类型匹配**：相同数据类型和用途

### 低优先级匹配
1. **关键词匹配**：包含相同关键词
2. **系统匹配**：相同的系统集成方向

## 故障排除

### 问题：找不到匹配结果
- 检查关键字是否准确
- 尝试使用同义词（采购订单→PO→PurchaseOrder）
- 检查对照清单文件是否存在

### 问题：匹配度低
- 补充需求清单的业务场景描述
- 添加更多字段信息
- 明确接口方向（T100→OA 或 OA→T100）

### 问题：脚本执行失败
- 确认已安装 openpyxl：`pip install openpyxl`
- 检查文件路径是否正确
- 查看错误日志

### 问题：本地文件不存在
- 访问在线文档：https://docs.qq.com/sheet/DWmF4RXZmdW5RWnVk?tab=nxp5uc
- 手动下载 Excel 文件到 `需求对照工具表/` 目录：
  - `【2026】字段对照---最小颗粒度.xlsx`（字段对照清单）
  - `【2026】接口清单---汇总.xlsx`（接口清单汇总）

### 问题：在线文档无法访问
- 检查网络连接
- 确认腾讯文档账号登录状态
- 验证文档访问权限（可能需要申请权限）
- 联系文档管理员获取访问权限

## AI 辅助匹配（新增）

当本地规则匹配无法找到合适的接口时，可以使用 AI 辅助匹配功能作为兜底方案。

### 功能说明

AI 辅助匹配使用 **Kimi API**（moonshot-v1-32k 模型）进行语义检索匹配：
- 自动提取需求中的业务实体和动作
- 在接口清单中进行语义相似度分析
- 返回最匹配的接口推荐及理由

### 前置条件

1. **安装依赖**（如果尚未安装）：
   ```bash
   npm install xlsx
   ```

2. **配置 Kimi API Key**：
   ```bash
   # 方式 1：环境变量
   set KIMI_API_KEY=sk-your-api-key

   # 方式 2：命令行参数
   node match_ai_assist.js xxx.xlsx --api-key sk-your-api-key
   ```

   获取 API Key：访问 https://platform.moonshot.cn/ 注册并创建

### 使用方式

#### 失败兜底模式（默认）
只分析本地规则匹配失败的行（匹配度 < 40%）：
```bash
node .qoder/skills/api-document-retrieval/scripts/match_ai_assist.js 需求评估整理/AI输入摸板.xlsx
```

#### 全量分析模式
对所有需求行都进行 AI 分析：
```bash
node .qoder/skills/api-document-retrieval/scripts/match_ai_assist.js 需求评估整理/AI输入摸板.xlsx --all
```

#### 高级选项
```bash
# 调整匹配阈值（默认 40）
node match_ai_assist.js xxx.xlsx --threshold 50

# 指定输出目录
node match_ai_assist.js xxx.xlsx --output 自定义输出目录/

# 指定模型
node match_ai_assist.js xxx.xlsx --model moonshot-v1-8k

# 查看所有选项
node match_ai_assist.js --help
```

### 输出文件

运行后会生成两个文件：

1. **AI辅助匹配报告** (`xxx_AI辅助匹配报告.md`)
   - 匹配统计（本地成功 / AI成功 / 仍未匹配）
   - AI 推荐接口详情（流程名称、匹配确信度、匹配理由）
   - 仍未匹配的需求列表

2. **JSON 数据文件** (`xxx_ai_matches.json`)
   - 每条需求的本地匹配结果和 AI 匹配结果
   - 便于后续程序化处理


**错误**：`[✗] AI 调用失败: ...`
**解决**：
- 检查网络连接
- 确认 API Key 有效
- 查看 Kimi 平台状态 https://status.moonshot.cn/
